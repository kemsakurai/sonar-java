式の複雑さが、演算子 &&, || and condition ? ifTrue : ifFalse の数により定義されています。

コードを読みやすくするために、単一の式が複雑になりすぎないようにします。

違反コード例

デフォルトの閾値は3です。

if (((condition1 && condition2) || (condition3 && condition4)) && condition5) { ... }

準拠方法

if ( (myFirstCondition() || mySecondCondition()) && myLastCondition()) { ... }

例外

equalsメソッドの内部では問題は報告されません。このメソッドの内部ではクラスのすべてのフィールドが等しいかどうかを比較するのが一般的だからです。
